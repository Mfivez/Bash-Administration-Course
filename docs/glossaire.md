# Glossaire

| Terme | Définition simple |
|---|---|
| Bash | Shell très utilisé sous Linux, utilisable en interactif et en scripts |
| Shell | Programme qui interprète les commandes de l'utilisateur |
| Terminal | Interface permettant d'interagir avec un shell |
| Script | Fichier contenant une suite de commandes exécutables |
| Shebang | Première ligne indiquant quel interpréteur lancer, ex. `#!/bin/bash` |
| Option | Paramètre qui modifie le comportement d'une commande, ex. `ls -l` |
| Argument | Valeur fournie à une commande, ex. `cat fichier.txt` |
| Pipe | Mécanisme qui transmet la sortie d'une commande à une autre, `|` |
| Redirection | Envoi de la sortie ou erreur vers un fichier |
| Permission | Droit de lire, écrire ou exécuter un fichier |
| Processus | Programme en cours d'exécution |
| Service | Processus système géré en arrière-plan, souvent par systemd |
| Log | Journal d'événements produit par un système ou une application |
| Cron | Planificateur de tâches récurrentes |
| Code retour | Nombre indiquant le succès ou l'échec d'une commande |
