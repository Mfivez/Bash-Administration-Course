# Contribuer au support

## Règles de rédaction

- Écrire en français clair.
- Garder les exemples courts et exécutables.
- Ne jamais proposer une commande destructive sans explication et garde-fou.
- Préférer une progression : notion, exemple, exercice, validation.
- Ajouter une correction quand un atelier est ajouté.

## Structure d'un module

Chaque journée doit contenir :

1. objectifs ;
2. notions clés ;
3. exemples commentés ;
4. exercices intégrés ;
5. atelier fil rouge ;
6. validation ;
7. points de vigilance ;
8. livrable.

## Vérification avant commit

```bash
bash tests/test_scripts.sh
```
